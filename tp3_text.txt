Partie 1: Introduction à Docker
 
1.1 Qu'est
-
ce que Docker?
 
Docker est une plateforme de virtualisation légère qui permet d'exécuter des 
applications dans des conteneurs isolés. Les conteneurs Docker sont autonomes et 
regroupent tout le nécessaire pour exécuter une application, y compris 
son code, ses 
dépendances, son système d'exploitation et ses configurations.
 
1.2 Images Docker
 
Une image Docker est un modèle réutilisable à partir duquel des conteneurs sont 
créés. Elle contient l'ensemble des instructions nécessaires pour configurer et
 
exécuter 
2
 
 
une application. Les images Docker peuvent être stockées localement sur votre 
machine ou dans des registres publics comme Docker Hub.
 
1.3 Conteneurs Docker
 
Un conteneur Docker est une instance exécutable d'une image Docker. Il s'agit d'un 
envir
onnement isolé qui encapsule l'application et ses dépendances. Les conteneurs 
Docker sont légers, portables et peuvent être démarrés, arrêtés et déplacés facilement.
 
1.4 Registres Docker
 
Un registre Docker est un référentiel centralisé pour stocker et pa
rtager des images 
Docker. Le registre Docker public le plus connu est Docker Hub, mais il existe 
également des registres privés pour les organisations et les entreprises.
 
1.5 Docker Compose
 
Docker Compose est un outil permettant de définir et d'exécuter des applications 
multi
-
conteneurs. Il simplifie la gestion de plusieurs conteneurs interdépendants en 
regroupant les configurations dans un seul fichier YAML.
